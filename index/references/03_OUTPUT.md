# Output Format

The generated `INDEX.md` follows this structure:

```markdown
# Skill Index

> Auto-generated by `index-skills`. Do not edit manually.
> Regenerate with: `.codex/skills/index-skills/scripts/index.sh`

## Quick Reference

| Skill | Path | Type |
|-------|------|------|
| `skill-name` | `path/to/skill/` | standalone \| skillset \| member |

## Skillsets

### `skillset-name`

> One-line description

**Path:** `path/to/skillset/`
**Members:** `member-a`, `member-b`, ...
**Default Pipeline:** member-a → member-b → ...

#### Members

- **`member-a`** — one-line description
- **`member-b`** — one-line description

---

(Repeat for each skillset)

## Standalone Skills

### `skill-name`

> One-line description

**Path:** `path/to/skill/`
**Keywords:** `keyword1`, `keyword2`, ...

---

(Repeat for each standalone skill)

## Keyword Index

| Keyword | Skills |
|---------|--------|
| `keyword` | `skill-a`, `skill-b` |
```

## Notes

- Descriptions are truncated to first sentence or 100 characters
- Keywords are de-duplicated and sorted alphabetically
- Skillset members are listed in pipeline order when available
- The index should be committed to version control
