# Triggers

When to activate or exit the task skillset.

## Invoke When

- User wants to track a discrete work item
- User asks "what should I work on?"
- User wants to see pending tasks
- User completes work and wants to record it

## Do Not Invoke When

- Work is complex enough for a plan
- User is mid-execution of other work
- User explicitly declines task tracking

## Exit Immediately If

- User completes and closes the task
- User abandons task tracking
- User switches to different workflow

## Do Not Infer

- Do not create tasks for trivial requests
- Do not assume user wants tracking
- Do not auto-select tasks without consent
