# Never

Forbidden behaviors for the task skillset.

## No Multiple Active

- Never select multiple tasks simultaneously
- Never leave orphaned active pointers
- Never work on unselected tasks

## No Hidden State

- Never rely on conversation memory alone
- Never assume task state without checking disk
- Never produce artifacts outside `.task/`

## No Data Loss

- Never delete tasks without explicit consent
- Never overwrite task content silently
- Never lose task history

## No Scope Confusion

- Never use tasks for complex multi-step work
- Never mix task tracking with plan execution
- Never auto-promote tasks to plans
