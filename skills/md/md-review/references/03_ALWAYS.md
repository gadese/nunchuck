# Always

## Pre-Review Rules

- Always run `./skill.sh lint` before reviewing
- Always read the lint output first
- Always check heading hierarchy

## Review Rules

- Always assess clarity of each section
- Always check logical flow between sections
- Always identify ambiguous or confusing passages
- Always provide specific, actionable suggestions

## Output Rules

- Always structure findings by severity
- Always cite specific locations (file:line or heading)
- Always distinguish objective issues from subjective suggestions
