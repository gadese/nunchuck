# Never

## Prohibited Behaviors

### Don't Skip Reading at Session Start
- **NEVER** skip reading Memory Bank files at the start of a session
- **NEVER** assume you know the current state without checking
- **NEVER** proceed with R-P-I workflows without reading Memory Bank first

### Don't Update for Minor Changes
- **NEVER** update Memory Bank for every small edit or minor change
- **NEVER** clutter files with trivial updates
- **NEVER** update just for the sake of updating

### Don't Leave Contradictory Information
- **NEVER** leave conflicting information across different files
- **NEVER** update one file without checking related files for consistency
- **NEVER** ignore discrepancies between files

### Don't Accumulate Stale Information
- **NEVER** let old, outdated information accumulate in files
- **NEVER** keep more than 2-3 sessions of history in `activeContext.md`
- **NEVER** preserve information that is no longer relevant

### Don't Modify File Structure
- **NEVER** change the 10-file structure of the Memory Bank
- **NEVER** rename core Memory Bank files
- **NEVER** add custom files without documenting them in `memory-index.md`

### Don't Skip Timestamps
- **NEVER** update a file without updating its "Last Updated" timestamp
- **NEVER** use inconsistent timestamp formats
- **NEVER** omit completion dates from finished features

### Don't Create Verbose Logs
- **NEVER** use Memory Bank as a detailed activity log
- **NEVER** include implementation details that belong in code comments
- **NEVER** duplicate information that exists elsewhere in the codebase

### Don't Update Without Reading First
- **NEVER** update Memory Bank files without reading them first
- **NEVER** overwrite existing content without understanding what's there
- **NEVER** assume the current state matches your expectations

### Don't Ignore Missing Files
- **NEVER** proceed if expected Memory Bank files are missing without initializing them
- **NEVER** create partial Memory Bank structures
- **NEVER** skip initialization when Memory Bank doesn't exist

### Don't Break Cross-References
- **NEVER** remove content that is referenced from other files without updating references
- **NEVER** create references to non-existent files or sections
- **NEVER** leave broken links in Memory Bank files

### Don't Skip changeLog
- **NEVER** skip changeLog entry when updating Memory Bank
- **NEVER** delete archives
- **NEVER** read changeLog by default at session start (only read when needed for history)

### Don't Over-Update codeInventory
- **NEVER** update codeInventory for implementation-only changes (internal logic changes that don't affect signatures)
