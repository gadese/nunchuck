# Triggers

## Invoke when

- The user requests a review/assessment of a completed plan.
- All plan tasks are marked complete and you need a final pass.

## Do not invoke when

- The user wants to execute tasks (use `plan-exec`).

## Exit immediately if

- The plan is not complete and the user wants execution instead of review.
