# Instructions

## Initialize

1. Read all reference files listed in `metadata.references` before taking action.
2. Identify the target plan using `./skill.sh status` or user specification.

## Policies

### Always

1. Review each task's Output section for concrete results.
2. Verify success criteria in the root plan.md are checkable.
3. Produce a summary assessment with strengths and gaps.

### Never

1. Never modify completed task files.
2. Never mark criteria as met without evidence.
3. Never skip tasks during review.
