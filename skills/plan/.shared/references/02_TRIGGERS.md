# Triggers

When to activate or exit the plan skillset.

## Invoke When

- User has multi-step work requiring structure
- Work spans multiple sessions or handoffs
- User wants trackable progress
- Work is complex enough to benefit from decomposition

## Do Not Invoke When

- Single-file changes
- Quick fixes or typos
- Work that fits in one conversation turn
- User explicitly wants informal approach

## Exit Immediately If

- User abandons the plan
- All tasks complete and review passes
- User switches to different work

## Do Not Infer

- Do not create plans for trivial requests
- Do not assume plan structure without discussion
- Do not expand scope beyond original objective
