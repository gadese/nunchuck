# 04_NEVER.md — Forbidden behaviors

**Purpose**:

* Explicitly define failure modes to avoid
* Prevent subtle drift or agent “helpfulness”

**Contains**:

* Disallowed actions
* Anti-patterns
* Known footguns

**Examples**:

* “Never invent files”
* “Never assume persistence”
* “Never modify user code without explicit instruction”

**Constraints**:

* Negative statements only
* No “unless”
* No suggestions

> This doc is disproportionately powerful. Keep it sharp.
